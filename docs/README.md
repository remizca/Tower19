# Documentation Structure

This folder contains all project documentation organized by category:

## Directory Structure

- `/docs` - Root documentation folder
  - `/generators` - Generator algorithms and specifications
    - `algorithms.md` - Core 3D generator algorithms and heuristics
  - `/progress` - Implementation tracking
    - `PROGRESS.md` - Detailed implementation progress and technical decisions
  - `/schema` - Data schemas and type definitions
    - `part-recipe.schema.json` - JSON Schema for part definitions
  - `/specs` - Technical specifications
    - `2d-drawing.md` - ISO-compliant 2D drawing specifications
    - `persistence.md` - Client-side storage specifications
    - `ui-interactions.md` - UI/UX interaction specifications

## Document Types

1. **Core Documentation**
   - High-level project goals and architecture
   - Setup and deployment guides
   - Contributing guidelines

2. **Technical Specifications**
   - Detailed technical requirements
   - Standards compliance documentation
   - API specifications

3. **Progress Tracking**
   - Detailed implementation status
   - Technical decisions and rationale
   - Sprint/milestone tracking

4. **Reference Documentation**
   - Data schemas
   - Type definitions
   - API documentation

## Documentation Guidelines

1. Keep documentation up to date with code changes
2. Use Markdown for all documentation
3. Include code examples where relevant
4. Cross-reference related documents
5. Keep each document focused on a single topic
